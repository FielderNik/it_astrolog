# IT Астролог

Веб-приложение, которое создаёт шуточный астрологический прогноз для IT-специалиста на выбранную дату. Прогноз учитывает профессию и знак зодиака, а в тексте обыгрывает рабочие достижения, типичные риски и фейлы.

На первом этапе приложение работает полностью в браузере и использует локальную языковую модель через Ollama. Архитектура должна позволить позднее подключить облачную модель через API без переработки экранов и сценария генерации.

## Что входит в первую версию

- Выбор IT-профессии из подготовленного списка.
- Выбор даты рождения и автоматическое определение знака зодиака.
- Выбор даты прогноза.
- Генерация одного короткого, доброжелательного и смешного прогноза на русском языке.
- Выбор AI-провайдера и модели в настройках: локальная модель обязательна для задания; облачный провайдер предусмотрен как расширение.
- Понятные состояния: модель недоступна, идёт генерация, прогноз готов, запрос завершился ошибкой.

Приложение не собирает аккаунты, не хранит данные на сервере и не является настоящей астрологической или профессиональной рекомендацией.

## Локальный запуск

Нужны Node.js 20+ и установленная [Ollama](https://ollama.com/).

```bash
npm install
cp .env.example .env.local
ollama pull llama3.2
npm run dev
```

Откройте адрес, который напечатает Vite (обычно `http://localhost:5173`). Если Ollama не запускается автоматически вместе с её приложением, выполните в отдельном терминале:

```bash
ollama serve
```

В настройках сайта можно изменить URL локального сервера и имя модели. Они сохраняются только в `localStorage` браузера. Дата рождения используется в браузере только для вычисления знака и в запрос к модели не передаётся.

Если браузер сообщает о CORS при локальной разработке, разрешите origin Vite в настройках Ollama через переменную `OLLAMA_ORIGINS`, затем перезапустите Ollama. Например, для адреса по умолчанию: `OLLAMA_ORIGINS=http://localhost:5173 ollama serve`.

## Публикация

После `npm run build` опубликуйте содержимое папки `dist/` на любом статическом хостинге (Netlify, Vercel, GitHub Pages или аналогичном). Публичный сайт сможет обращаться только к Ollama на компьютере самого посетителя; облачный провайдер пока представлен архитектурной заготовкой и не принимает ключи в сборку.

## Статус

Реализован первый пользовательский путь: форма → вычисление знака → Ollama → прогноз или понятная ошибка. Настройки провайдера и модели хранятся локально; облачный адаптер оставлен как безопасная заготовка за тем же контрактом.
