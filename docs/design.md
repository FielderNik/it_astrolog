# Дизайн-система IT Астролога

## Визуальное направление

Основа — тёмная сцена с большим количеством воздуха: чёрный фон, белая крупная типографика, один яркий фиолетовый сигнал действия и редкие янтарные акценты. Направление вдохновлено стилем [Dala на Refero](https://styles.refero.design/style/e5f5f8cf-e68d-4ed1-bbf5-6b67569af648): свободная композиция, чистый чёрный фон, крупные заголовки и минимальное число заполненных элементов.

В IT Астрологе эта эстетика становится не интерфейсом «про знания», а интерфейсом «космической смены IT-специалиста»: лёгкие созвездия и точки фона намекают на гороскоп, а фиолетовый цвет отмечает точку действия. Это развлекательное, чуть ироничное настроение, но не неоновый киберпанк и не эзотерический салон.

Не переносить из референса фирменное название, логотип, композицию с мозгом из частиц, тексты или изображения. Декоративная графика приложения должна быть собственной и абстрактной.

## Принципы

- **Воздух важнее контейнеров.** Экран строится из типографики и отступов, а не из набора карточек.
- **Одно главное действие.** На форме и на экране результата одновременно допускается ровно одна залитая фиолетовая кнопка.
- **Цвет сообщает роль.** Фиолетовый — действие, янтарный — акцент или статус, белый — содержание, серый — вторичная информация.
- **Форма остаётся удобной.** Художественная минималистичность не отменяет заметные подписи, границы полей, видимый фокус и сообщения об ошибке.
- **Декор не мешает чтению.** Созвездия и точки не несут смысла, имеют `aria-hidden="true"`, не анимируются постоянно и отключаются при `prefers-reduced-motion`.

## Токены

### Цвет

| Роль | Токен | Значение | Применение |
| --- | --- | --- | --- |
| Фон | `--color-void` | `#000000` | Холст страницы и широкие свободные зоны. |
| Основной текст | `--color-bone` | `#FFFFFF` | Заголовки, значения полей, прогноз. |
| Вторичный текст | `--color-muted` | `#BDBDBD` | Пояснения, подписи, дата генерации. |
| Тихий текст | `--color-subtle` | `#9A9A9A` | Навигация, неактивные действия, плейсхолдеры. |
| Главное действие | `--color-iris` | `#8052FF` | Единственная залитая кнопка и её hover-состояние. |
| Акцент | `--color-saffron` | `#FFB829` | Слово-акцент в заголовке, успешный статус, фокус. |
| Дополнительный декор | `--color-verdant` | `#15846E` | Редкие точки/линии созвездия; не использовать как основной текст. |
| Поверхность поля | `--color-field` | `#111114` | Только фон интерактивных полей и компактных служебных блоков. |
| Граница поля | `--color-stroke` | `#3A3A40` | Граница и разделители. |
| Ошибка | `--color-danger` | `#FF7A90` | Текст ошибки и контур ошибочного поля. |

Цвета `#000000`, `#8052FF`, `#FFB829`, `#15846E`, `#9A9A9A`, `#BDBDBD` и `#FFFFFF` взяты из референса. Новые `field`, `stroke` и `danger` добавлены для различимости формы и ошибок.

### Типографика

Референс использует PP Neue Montreal. Не подключать этот шрифт без лицензии. Базовый безопасный стек проекта: `Inter, Arial, sans-serif`; если лицензия будет предоставлена, PP Neue Montreal можно поставить первым в стек без изменения шкалы.

| Роль | Токен | Desktop | Mobile | Начертание и интерлиньяж |
| --- | --- | --- | --- | --- |
| Display | `--text-display` | `clamp(52px, 7vw, 78px)` | 48–56px | 400 / 1.02, `letter-spacing: -0.04em` |
| H1 | `--text-h1` | 48px | 36px | 400 / 1.1, `-0.035em` |
| H2 | `--text-h2` | 32px | 27px | 400 / 1.15, `-0.02em` |
| Прогноз | `--text-forecast` | 24px | 22px | 400 / 1.35 |
| Основной текст | `--text-body` | 18px | 16px | 400 / 1.5 |
| Поле и кнопка | `--text-control` | 16px | 16px | 500 / 1.3 |
| Подпись | `--text-label` | 12–14px | 12–14px | 600 / 1.2, uppercase, `0.025em` |

Не использовать начертание `200` для рабочего текста формы: на чёрном фоне оно ухудшит читаемость. Лёгкость референса здесь достигается свободным пространством, а не снижением контраста текста.

## Компоновка

### Страница прогноза

На десктопе используется асимметричная сетка из двух колонок: в левой — контекст и форма, в правой — атмосферный заголовок и затем результат. Первая видимая область должна сразу объяснять сценарий и давать возможность начать.

```text
┌──────────────────────────────────────────────────────────────────┐
│ IT АСТРОЛОГ                                 Настройки модели     │
│                                                                  │
│ Твой релиз сегодня       ┌────────────────────────────────────┐ │
│ под счастливой           │  ПРОФЕССИЯ                         │ │
│ звездой.                 │  [ Frontend-разработчик          ] │ │
│                           │                                    │ │
│ Короткое пояснение.       │  ДАТА РОЖДЕНИЯ                     │ │
│                           │  [ 14.04.1997                    ] │ │
│                           │  Овен                              │ │
│                           │                                    │ │
│                           │  ДАТА ПРОГНОЗА                     │ │
│                           │  [ 12.07.2026                    ] │ │
│                           │                                    │ │
│                           │  [ Узнать прогноз ]                │ │
│                           └────────────────────────────────────┘ │
│                                                                  │
│ ─────────────── результат после генерации ───────────────────── │
│  Овен, сегодня твой деплой…                         [Повторить] │
└──────────────────────────────────────────────────────────────────┘
```

На экране до 767px колонки складываются: шапка → заголовок → форма → результат. Не использовать горизонтальную прокрутку, мелкий display-текст или закреплённые декоративные элементы поверх полей.

### Оболочка

- Контент центрируется в пределах 1120px и имеет боковые отступы 24px на desktop, 18px на mobile.
- Шапка прозрачная, без нижней линии: название слева, текстовая кнопка «Настройки модели» справа.
- Форма — единственный функциональный «остров». Она может иметь фон `--color-field`, радиус 24px и тонкую границу `--color-stroke`; никаких теней, стеклянных эффектов или нескольких вложенных карточек.
- Результат не помещается в тяжёлую карточку: его отделяет большой отступ, короткая декоративная линия и яркий типографический блок.

## Компоненты и состояния

### Заголовочный блок

- Малый ярлык: «ПРОГНОЗ ДЛЯ IT-ШНИКА», `--text-label`, янтарный.
- H1: 2–4 строки, белый display; одно ключевое слово разрешено выделить янтарным.
- Подзаголовок: до 2 строк, `--color-muted`.
- Не использовать больше одного декоративного акцента в заголовке.

### Поле формы

- Каждое поле имеет постоянную текстовую `label` над контролом; плейсхолдер её не заменяет.
- Селект профессии и поля дат имеют высоту от 48px, фон `--color-field`, 1px границу `--color-stroke`, радиус 12px.
- При фокусе: 2px внешний контур `--color-saffron` и сохранение контраста текста.
- При ошибке: граница и короткий текст под полем — `--color-danger`; не полагаться только на цвет.
- Вычисленный знак зодиака показывается под датой рождения как спокойный текстовый статус, а не как вводимое поле.

### Основная кнопка

- Текст: «Узнать прогноз»; в состоянии ожидания — «Сверяем звёзды и логи…».
- Фон `--color-iris`, белый текст, pill-радиус, высота от 48px.
- Кнопка занимает ширину формы на mobile и имеет естественную ширину на desktop.
- Hover — повышение яркости, active — небольшое затемнение; disabled — серый фон и курсор по умолчанию.
- Рядом не должно быть второй залитой кнопки. «Сбросить» и «Настройки модели» — текстовые действия.

### Блок прогноза

- До результата: спокойная строка «Прогноз появится здесь после запроса к модели» в `--color-subtle`.
- Загрузка: три небольшие точки или тонкая мерцающая линия; анимация выключается при `prefers-reduced-motion`.
- Готовый прогноз: малый янтарный ярлык с профессией и знаком, затем текст `--text-forecast`; дата и модель — вторичные данные ниже.
- Ошибка: заголовок «Не удалось достучаться до звёздной модели», короткое действие по исправлению и текстовая кнопка повтора. Сообщение не должно звучать как ошибка пользователя.

### Настройки модели

- Открываются по текстовой кнопке в шапке; на desktop — компактная панель/диалог, на mobile — bottom sheet.
- Содержит выбор провайдера, базовый URL, модель и индикатор доступности.
- Это служебный блок: без яркой иллюстрации и без конкуренции с прогнозом.
- Если выбран облачный провайдер, рядом показывается, что режим пока недоступен. Поле для API-ключа не выводится.

## Декоративный слой

- Разрешены: 12–30 редких точек и тонких линий-созвездий, 2–3 цвета из `iris`, `saffron`, `verdant`, прозрачность до 45%.
- Запрещены: фотосток, «магический» хрустальный шар, пентаграммы, анимация на весь экран, копирование нейронного облака референса.
- Декор расположен только по внешним краям композиции и никогда не перекрывает поля, текст прогноза или кнопку.
- На мобильном остаётся не более 6–10 точек; при снижении движения декоративная анимация не запускается.

## Доступность

- Контраст текста и интерактивных элементов проверяется по WCAG AA: обычный текст не ниже 4.5:1, крупный — 3:1.
- Не использовать цвет как единственный носитель состояния.
- У всех полей есть `label`, у ошибок — связанный с полем текст, у области результата — доступное сообщение о загрузке.
- Видимый фокус обязателен и не должен исчезать на тёмном фоне.
- Основные действия доступны с клавиатуры; порядок табуляции соответствует визуальному порядку формы.
- Пользователь может прочитать и заполнить форму при масштабе браузера 200%.

## Что не делать

- Не заменять чёрный фон градиентом или набором серых карточек.
- Не делать фиолетовый цвет фоном всей секции или каждого элемента.
- Не использовать более одной акцентной кнопки в пределах одного сценария.
- Не злоупотреблять свечением, тенью, glassmorphism и неоновыми рамками.
- Не прятать подсказки, ошибки или статус модели только в иконке.
- Не перегружать экран знаками зодиака, эмодзи и «мистическими» клише.

## Перед реализацией

1. Согласовать тексты первого экрана и точный список IT-профессий.
2. Получить лицензию на PP Neue Montreal либо оставить Inter основным шрифтом.
3. Сделать простой чёрно-белый wireframe и проверить порядок формы на mobile.
4. Реализовать токены и один основной экран, затем проверить контраст и клавиатурную навигацию.
