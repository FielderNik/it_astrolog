# Инструкции для Codex

## Контекст

`IT Астролог` — статическое веб-приложение на русском языке. Пользователь выбирает IT-профессию, дату рождения и дату прогноза; приложение определяет знак зодиака и просит LLM написать шуточный прогноз, связанный с профессией и знаком.

Основной обязательный путь — локальная модель Ollama. Облачные LLM подключаются только через общий слой провайдера, чтобы не дублировать логику экранов и промптов.

## Границы первой версии

- Только веб-клиент; не добавлять сервер, базу данных, авторизацию или аналитику без явного запроса.
- Перед изменением интерфейса читать `docs/design.md`. Не добавлять новый визуальный язык, UI-kit или декоративные элементы в обход зафиксированных токенов и правил.
- Не превращать прогноз в медицинский, финансовый, юридический или кадровый совет.
- Все пользовательские тексты и Markdown-документация — на русском языке. Английский допустим только в именах библиотек, переменных, API и командах.

## Технические правила

- Использовать TypeScript в строгом режиме.
- Держать доменную логику отдельно от React-компонентов: определение знака, валидация формы, сборка промпта и нормализация ответа не должны жить в JSX.
- Обращение к LLM выполнять только через интерфейс провайдера. Экран генерации не должен зависеть от URL, формата запроса или названия модели.
- Локальные настройки и последний удачный ввод можно хранить только в `localStorage` и без персональных идентификаторов.
- Не добавлять ключи, токены или реальные `.env.local` в Git. Публичный фронтенд не может безопасно хранить ключ облачного API.

## Проверка изменений

После изменения логики запускать профильные unit-тесты. Перед завершением задачи запускать как минимум typecheck/build и сообщать, если проверку выполнить не удалось. Новые сценарии, меняющие пользовательское поведение, должны иметь тесты для чистых функций.

## Минимальная структура после старта разработки

```text
src/
  app/          # сборка приложения и провайдеры состояния
  domain/       # профессии, зодиак, типы прогноза, валидация
  features/     # форма и показ прогноза
  llm/          # интерфейс и реализации AI-провайдеров
  lib/          # чистые общие утилиты
```

Не вводить сложную архитектуру, глобальный state manager или дополнительные слои, пока для них нет конкретной потребности.
